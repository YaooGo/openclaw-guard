# Contributing to OpenClaw Guard

Thank you for your interest in contributing to OpenClaw Guard! We welcome contributions from the community to help make AI programming safer.

## How to Contribute

1. **Reporting Bugs**: Please use GitHub Issues to report bugs. Include steps to reproduce and your environment details.
2. **Feature Requests**: Open an issue to discuss new features before implementing them.
3. **Pull Requests**:
   - Fork the repository.
   - Create a feature branch.
   - Ensure your code follows the project's style (Go for backend, Vue for frontend).
   - Write or update tests if applicable.
   - Submit a PR with a clear description of changes.

## Development Setup

See the [README.md](README.md) for detailed instructions on setting up the backend and frontend development environments.

## Security

If you discover a security vulnerability, please do NOT open a public issue. Instead, contact the maintainers directly or use the GitHub security advisory feature if enabled.

# 参与贡献

感谢您对 OpenClaw Guard 的关注！我们欢迎社区贡献，共同提升 AI 编程的安全性。

## 贡献方式

1. **报告 Bug**：请使用 GitHub Issues 提交 Bug。请包含重现步骤和环境信息。
2. **功能建议**：在实现新功能前，请先开 Issue 讨论。
3. **提交 PR**：
   - Fork 本仓库。
   - 创建功能分支。
   - 确保代码符合项目规范（后端 Go，前端 Vue）。
   - 如果适用，编写或更新测试。
   - 提交 PR 并简要说明修改内容。

## 开发设置

请参考 [README.md](README.md) 了解如何搭建后端和前端开发环境。

## 安全漏洞

如果您发现安全漏洞，请**不要**公开发布 Issue。请通过私密渠道联系维护者。
